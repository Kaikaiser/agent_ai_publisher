# 前端运行逻辑文档

## 1. 这份文档解决什么问题

这份文档专门解释本项目当前前端代码的运行逻辑，包括：

- 前端是怎么启动的
- 页面路由怎么工作
- 登录态怎么保存
- 登录、问答、知识导入、历史页面分别怎么运行
- 前端如何调用后端 API
- 当前前端设计的优点与限制

## 2. 前端入口从哪里开始

前端入口在：

- `frontend/src/main.jsx`

Vite 启动命令通常是：

```bash
cd frontend
npm run dev
```

启动后，浏览器会加载：

- `frontend/index.html`
- `frontend/src/main.jsx`

`main.jsx` 的作用是：

1. 挂载 React 应用
2. 用 `BrowserRouter` 包裹整个应用
3. 渲染 `App` 组件

也就是说，真正的页面路由和页面切换逻辑，从 `App.jsx` 开始。

## 3. 前端整体结构怎么分工

当前前端主要分为三类代码：

### 3.1 页面层

目录：

- `frontend/src/pages`

页面包括：

- `LoginPage.jsx`
- `ChatPage.jsx`
- `KnowledgePage.jsx`
- `HistoryPage.jsx`

这些文件负责：

- 页面布局
- 表单交互
- 触发 API 请求
- 渲染后端返回结果

### 3.2 接口层

目录：

- `frontend/src/api`

当前核心文件：

- `client.js`

它负责：

- 拼接后端 API 地址
- 自动附带 token
- 统一处理请求和错误

### 3.3 登录态层

目录：

- `frontend/src/auth`

当前核心文件：

- `session.js`

它负责：

- 保存 token
- 保存当前用户信息
- 读取当前用户
- 清除登录状态

## 4. 页面路由是怎么工作的

代码位置：

- `frontend/src/App.jsx`

当前前端使用：

- `react-router-dom`

路由设计分两层：

### 4.1 公共路由

当前公共路由只有：

- `/login`

它对应 `LoginPage`。

### 4.2 受保护路由

其余页面都放在 `ProtectedLayout` 下面。

这个组件会先做一件事：

- 读取当前用户

如果没有用户信息，就会：

- 跳回 `/login`

这就是当前前端最基础的“路由守卫”逻辑。

### 4.3 角色控制

当前角色控制写在 `ProtectedLayout` 里。

例如：

- 普通用户可以看问答页和历史页
- 只有管理员可以看到“知识导入”页

这是一种非常轻量但足够清晰的前端权限实现方式。

## 5. 登录页面是怎么运行的

代码位置：

- `frontend/src/pages/LoginPage.jsx`

运行逻辑如下：

1. 页面初始化时维护本地状态：
   - `username`
   - `password`
   - `error`
2. 如果已经存在登录用户，则直接跳转到 `/chat`
3. 用户点击“登录”按钮后：
   - 调用 `/auth/login`
4. 登录成功后：
   - 保存 token 和用户信息
   - 跳转到 `/chat`
5. 登录失败后：
   - 在页面上显示错误信息

当前默认填写的登录信息是：

- `admin`
- `admin123456`

这样做的目的主要是方便本地演示。

## 6. 登录态是怎么保存的

代码位置：

- `frontend/src/auth/session.js`

当前有三个函数：

- `saveSession(payload)`
- `getCurrentUser()`
- `logout()`

### 6.1 `saveSession`

登录成功后会把：

- `access_token`
- `user`

写入 `localStorage`。

### 6.2 `getCurrentUser`

页面渲染时会从 `localStorage` 读取用户信息，用于判断：

- 当前是否已登录
- 当前角色是什么

### 6.3 `logout`

退出登录时会删除本地存储中的：

- `token`
- `user`

然后页面跳转到登录页。

## 7. 前端 API 请求是怎么工作的

代码位置：

- `frontend/src/api/client.js`

核心常量：

- `API_BASE_URL = 'http://127.0.0.1:8000/api'`

核心函数：

- `apiRequest(path, options)`

它的作用是统一处理请求细节：

1. 从 `localStorage` 里读取 token
2. 自动设置请求头
3. 如果不是 `FormData`，自动补 `Content-Type: application/json`
4. 如果有 token，自动带上 `Authorization`
5. 请求失败时，统一抛出错误

所以页面层不需要每次都手动拼完整 URL 或处理鉴权头。

## 8. 问答页面怎么运行

代码位置：

- `frontend/src/pages/ChatPage.jsx`

运行流程如下：

1. 页面维护本地状态：
   - `question`
   - `bookTitle`
   - `docType`
   - `result`
   - `error`
2. 用户输入问题并点击发送
3. 页面调用：
   - `POST /api/chat/ask`
4. 请求体里包含：
   - 问题
   - 可选书名过滤
   - 可选文档类型过滤
5. 后端返回后，前端把结果保存到 `result`
6. 页面渲染：
   - 回答内容
   - 是否有知识依据
   - 引用来源列表

这意味着前端问答页本身不做任何智能处理，它只是把后端返回的结构化结果展示出来。

## 9. 知识导入页面怎么运行

代码位置：

- `frontend/src/pages/KnowledgePage.jsx`

运行流程如下：

1. 管理员选择文件
2. 输入：
   - 书名
   - 文档类型
   - 可见角色
3. 构造 `FormData`
4. 调用：
   - `POST /api/knowledge/import`
5. 导入成功后，显示结果消息
6. 导入失败后，显示错误信息

这个页面的关键点是：

- 请求体不是 JSON，而是 `FormData`
- 页面本身并不解析文件，只负责把文件上传给后端

## 10. 历史记录页面怎么运行

代码位置：

- `frontend/src/pages/HistoryPage.jsx`

运行流程如下：

1. 页面加载时触发 `useEffect`
2. 调用：
   - `GET /api/conversations`
3. 把返回的数据保存到 `items`
4. 用列表形式渲染历史记录

当前页面展示的重点是：

- 时间
- 问题
- 回答

它没有展开所有来源细节，保持了页面简洁。

## 11. 页面布局怎么工作的

代码位置：

- `frontend/src/App.jsx`
- `frontend/src/styles.css`

### 11.1 布局结构

当前使用一个简单的后台式布局：

- 左侧导航栏
- 右侧主内容区域

左侧导航栏负责：

- 显示项目名称
- 显示当前用户信息
- 显示导航链接
- 提供退出登录按钮

### 11.2 样式策略

当前样式是轻量手写 CSS，不依赖 UI 框架。

这样设计的好处：

- 更容易读懂
- 没有额外组件库心智负担
- 适合首版快速迭代

## 12. 一个完整的前端运行流程

你可以把前端整体理解成下面这条链路：

### 启动时

1. Vite 启动开发服务器
2. 浏览器加载 `main.jsx`
3. React 渲染 `App`
4. 路由系统开始工作

### 用户进入系统时

1. 如果没有登录信息，进入 `/login`
2. 输入用户名密码
3. 调用后端登录接口
4. 保存 token 和用户信息
5. 跳转到 `/chat`

### 用户提问时

1. 问答页收集输入
2. 调用问答接口
3. 拿到后端返回结果
4. 展示回答与来源

### 管理员导入知识时

1. 导入页上传文件
2. 后端执行导入与索引构建
3. 前端展示导入结果

### 用户查看历史时

1. 历史页请求历史接口
2. 展示当前用户的问答记录

## 13. 当前前端设计的优点

- 简单直接，适合新手理解
- 页面的职责清晰
- 登录态逻辑集中
- API 请求逻辑集中
- 与当前后端能力一一对应
- 没有过早引入复杂状态管理

## 14. 当前前端的限制

- 错误提示还不够细
- token 直接放在 localStorage，安全性一般
- 没有全局 loading 管理
- 没有统一的 UI 组件层
- 没有请求重试与刷新 token 机制
- 没有更细粒度权限页面策略

## 15. 后续前端优化建议

推荐优先级如下：

1. 优化错误提示
- 区分网络错误、登录失败、后端未启动、权限错误

2. 增强问答体验
- 增加 loading 状态
- 增加引用来源折叠展示
- 增加复制回答功能

3. 增加用户管理页面
- 管理员创建用户
- 用户列表

4. 优化登录态
- token 过期处理
- 自动退出
- 更安全的会话方案

5. 组件化整理
- 抽出表单组件
- 抽出导航组件
- 抽出卡片组件

## 16. 推荐阅读顺序

如果你准备看前端代码，建议顺序是：

1. `frontend/src/main.jsx`
2. `frontend/src/App.jsx`
3. `frontend/src/auth/session.js`
4. `frontend/src/api/client.js`
5. `frontend/src/pages/LoginPage.jsx`
6. `frontend/src/pages/ChatPage.jsx`
7. `frontend/src/pages/KnowledgePage.jsx`
8. `frontend/src/pages/HistoryPage.jsx`
9. `frontend/src/styles.css`

这个顺序基本就是：

- 先看怎么启动
- 再看路由
- 再看登录态
- 再看接口
- 最后看各个页面
