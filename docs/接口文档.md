# API 接口文档

## 1. 说明

本文档描述“出版社 AI 助手”当前后端对外提供的 HTTP API。

当前服务基地址：

```text
http://127.0.0.1:8000
```

接口前缀：

```text
/api
```

认证方式：

- Bearer Token
- 登录成功后，把 `access_token` 放入请求头：

```text
Authorization: Bearer <token>
```

## 2. 通用约定

### 2.1 Content-Type

- JSON 接口：`application/json`
- 文件上传接口：`multipart/form-data`

### 2.2 认证错误

常见错误：

- `401 Unauthorized`
  - token 缺失
  - token 无效
  - 用户不存在
- `403 Forbidden`
  - 当前用户没有管理员权限

## 3. 健康检查

### 3.1 获取服务状态

- 方法：`GET`
- 路径：`/health`

示例响应：

```json
{
  "status": "ok"
}
```

## 4. 认证接口

### 4.1 登录

- 方法：`POST`
- 路径：`/api/auth/login`
- 是否需要认证：否

请求体：

```json
{
  "username": "admin",
  "password": "admin123456"
}
```

成功响应：

```json
{
  "access_token": "<jwt>",
  "token_type": "bearer",
  "user": {
    "username": "admin",
    "role": "admin"
  }
}
```

失败响应示例：

```json
{
  "detail": "Invalid username or password"
}
```

## 5. 问答接口

### 5.1 提交文本问题

- 方法：`POST`
- 路径：`/api/chat/ask`
- 是否需要认证：是

请求体：

```json
{
  "question": "这本教辅适合几年级？",
  "book_title": "小学数学",
  "doc_type": "textbook"
}
```

成功响应：

```json
{
  "conversation_id": 1,
  "answer": "根据知识库，这本教辅适合三年级。",
  "grounded": true,
  "sources": [
    {
      "content": "本书适合三年级学生，覆盖加减乘除...",
      "filename": "sample_textbook.txt",
      "book_title": "小学数学",
      "doc_type": "textbook",
      "location": "full-text"
    }
  ]
}
```

说明：

- 如果知识库没有找到充分依据，系统会返回保守回答
- 如果未配置 `OPENAI_API_KEY`，启动虽可成功，但问答会失败

## 6. 拍照搜题接口

### 6.1 识别图片中的题目文本

- 方法：`POST`
- 路径：`/api/chat/recognize-image`
- 是否需要认证：是
- 请求方式：`multipart/form-data`

表单字段：

- `file`: 题目图片，必填

成功响应：

```json
{
  "recognized_text": "已识别题目：1+1等于几？"
}
```

说明：

- 这个接口只做 OCR，不直接问答
- 适合“两步式拍照搜题”第一步

### 6.2 图片直接识别并问答

- 方法：`POST`
- 路径：`/api/chat/ask-image`
- 是否需要认证：是
- 请求方式：`multipart/form-data`

表单字段：

- `file`: 题目图片，必填
- `book_title`: 书名过滤，可选
- `doc_type`: 文档类型过滤，可选

成功响应：

```json
{
  "conversation_id": 2,
  "recognized_text": "已识别题目：1+1等于几？",
  "answer": "根据识别结果，答案是 2。",
  "grounded": true,
  "sources": [
    {
      "content": "1+1=2。",
      "filename": "math.txt",
      "book_title": "基础数学",
      "doc_type": "exercise",
      "location": "full-text"
    }
  ]
}
```

说明：

- 这是“一步式图片问答”接口
- 当前前端主要推荐走“两步式”：先 `recognize-image`，再 `ask`
- 如果图片不是图片类型或为空，会返回 `400`

## 7. 知识导入接口

### 7.1 导入文件

- 方法：`POST`
- 路径：`/api/knowledge/import`
- 是否需要认证：是
- 角色要求：`admin`
- 请求方式：`multipart/form-data`

表单字段：

- `file`: 上传文件
- `book_title`: 书名
- `doc_type`: 文档类型
- `allowed_role`: 可见角色，默认 `user`

成功响应：

```json
{
  "job_id": 1,
  "status": "completed",
  "message": "Indexed 1 chunks"
}
```

## 8. 历史记录接口

### 8.1 获取当前用户历史记录

- 方法：`GET`
- 路径：`/api/conversations`
- 是否需要认证：是

成功响应：

```json
{
  "items": [
    {
      "id": 1,
      "question": "这本教辅适合几年级？",
      "answer": "根据知识库，这本教辅适合三年级。",
      "grounded": true,
      "created_at": "2026-04-01T10:00:00"
    }
  ]
}
```

说明：

- 只返回当前登录用户的历史记录
- 当前版本不会把历史自动注入后续对话上下文

## 9. 错误码建议理解

虽然当前项目还没有统一错误码系统，但可以先按 HTTP 状态码理解：

- `200`: 成功
- `400`: 参数错误或业务处理失败
- `401`: 未登录或 token 无效
- `403`: 权限不足
- `500`: 服务内部异常

## 10. 推荐联调顺序

推荐的接口联调顺序：

1. `GET /health`
2. `POST /api/auth/login`
3. `POST /api/knowledge/import`
4. `POST /api/chat/recognize-image`
5. `POST /api/chat/ask`
6. `POST /api/chat/ask-image`
7. `GET /api/conversations`
